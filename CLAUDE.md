## Architecture

### Route Groups (critical layout structure)

- **`(frontend)/`** ŌĆö Public pages. Its `layout.tsx` owns `<html>` and `<body>` tags with Geist fonts and global CSS.
- **`(payload)/admin/`** ŌĆö Payload CMS admin panel. Its auto-generated `layout.tsx` uses `RootLayout` from `@payloadcms/next/layouts` which renders its own `<html>`/`<body>`.
- **Root `layout.tsx`** ŌĆö Returns `<>{children}</>` only (no `<html>`/`<body>`). This is intentional to avoid nested HTML conflicts between the two route groups.

### ISR + Revalidation Flow

Pages use ISR with `revalidate = 60`. On-demand revalidation via Payload hooks:

```
Payload afterChange hook ŌåÆ POST /api/revalidate (with x-revalidate-secret header)
ŌåÆ revalidatePath('/') + revalidatePath('/posts/[slug]', 'page')
```

### Image Loading (OSS)

Payload stores media metadata in the `media` collection; actual files live in Alibaba Cloud OSS. A custom image loader in `src/lib/image-loader.ts` converts Payload URLs to OSS virtual-hosted style with on-the-fly resize:

```
Payload URL:  /api/media/file/big.webp
  ŌåÆ OSS URL:  https://{bucket}.{endpoint}/big.webp?x-oss-process=image/resize,w_{width}
```

### Payload CMS Setup

- **Config**: `src/payload.config.ts` ŌĆö PostgreSQL adapter, Lexical editor, `@payloadcms/storage-s3` plugin for OSS, 3 collections (users, posts, media).
- **API route**: `src/app/api/[...slug]/route.ts` ŌĆö Auto-generated, do not edit.
- **Admin layout**: `src/app/(payload)/admin/[[...segments]]/layout.tsx` ŌĆö Auto-generated, imports `@payloadcms/next/css` for admin styles.
- **importMap**: `src/app/(payload)/admin/importMap.ts` ŌĆö Maps Payload component paths to imports. `RenderServerComponent` must be imported from `@payloadcms/ui/elements/RenderServerComponent` (not `@payloadcms/ui` directly).

### Data Fetching

`src/lib/posts.ts` wraps Payload queries with `unstable_cache` (tags: `['posts']`). `getAllPostSlugs` is used by `generateStaticParams()` for static generation.

### Styling

- **Tailwind CSS 4** via `@tailwindcss/postcss` (no `tailwind.config.*` file needed).
- **shadcn/ui** components in `src/components/ui/` ŌĆö uses `@base-ui/react` (NOT Radix). Button does NOT support `asChild` prop; use `buttonVariants()` + `Link` pattern instead.
- **Payload admin styles**: Loaded via `import '@payloadcms/next/css'` in the admin layout. Do NOT use SCSS imports for Payload styles.
- **CSS files**: `src/app/globals.css` and `src/app/(frontend)/globals.css` (both needed ŌĆö root and frontend layout respectively).

### Key Gotchas

- **pnpm strict mode**: `@payloadcms/ui` must be listed as a direct dependency, otherwise it's not resolvable from project code.
- **Payload CLI commands fail with ESM error**: `payload generate:types`, `payload generate:importmap` etc. crash with `ERR_REQUIRE_ASYNC_MODULE` on Node 20/22. Type generation and importMap generation must be done manually or via custom scripts.
- **Schema changes**: Adding required columns to existing tables triggers a confirmation prompt. Use `PAYLOAD_DROP_DATABASE=true` env var to reset during development.
- **Media uploads**: Uses `@payloadcms/storage-s3` plugin with Alibaba Cloud OSS. Files stored remotely, not on local disk.

### Path Aliases

- `@/*` ŌåÆ `./src/*`
- `@payload-config` ŌåÆ `./src/payload.config.ts`

### Docker

- `docker/production/` ŌĆö production self-contained directory (Dockerfile, compose, nginx, certs, 502 page).
  - `Dockerfile` ŌĆö multi-stage build with standalone output, pnpm store cache mount.
  - `compose.yaml` ŌĆö app + nginx services, volume paths reference files in same directory.
  - `nginx.conf` ŌĆö HTTPS reverse proxy with SSL termination, HTTPŌåÆHTTPS redirect, CSRF fix for Payload, 502 maintenance page.
  - `502.html` ŌĆö maintenance page shown during backend failures.
  - `certs/` ŌĆö SSL certificates (gitignored, auto-generated by `scripts/generate-certs.sh` if missing).
- `docker/development/` ŌĆö development self-contained directory.
  - `Dockerfile` ŌĆö development image with `WATCHPACK_POLLING=true` for container HMR.
  - `compose.yaml` ŌĆö dev service with watch mode, build context is project root (`../..`).
- `pnpm-workspace.yaml` needs `packages: ['.']` temporarily during `pnpm install` in build context.

### CI/CD

GitHub Actions workflow (`.github/workflows/deploy.yml`): push to `main` ŌåÆ build Docker image ŌåÆ push to ACR ŌåÆ SSH to ECS ŌåÆ pull & restart. Uses BuildKit GHA cache for layer caching.
